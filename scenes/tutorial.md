# Scene: Tutorial

## Goal

帮助受众从不了解走到能解释、能操作或能复现。成功标准是用户能完成明确任务，而不只是“看懂页面”。

## Required Input

主题、受众基础、最终能力、前置条件、步骤、实例、可能失败点。

## Information Structure

1. **Hero**：一句话说明学会什么、适合谁、预计成本。
2. **Context**：为什么需要它，给出问题或最终效果。
3. **Prerequisites**：工具、版本、素材与限制。
4. **Step Flow**：3-7 个有动作动词的步骤。
5. **Worked Example**：完整输入、过程与输出。
6. **Troubleshooting**：常见错误、症状和修复。
7. **Summary / Next Step**：复盘要点与下一行动。

## Recommendations

- Layout：`hero`、`step-flow`、`two-column`、`comparison`。
- Component：`annotation`、`code-block`、`terminal-window`、`prompt-block`、`badge`。

## Constraints

- 一个步骤只包含一个主要动作；依赖关系明确。
- 代码、命令和 Prompt 必须可复制，示例不得省略关键输入。
- 移动端按执行顺序单列排列，不能把关键说明藏在 hover 中。

## Common Errors

- 先讲原理很久才出现目标。
- 步骤只有名词，没有动作和结果。
- 只展示成功路径，不说明前置条件和失败状态。
- 示例与步骤使用不同数据，无法对照。

## Delivery Check

受众能定位当前步骤、预期结果、失败处理和下一步。
