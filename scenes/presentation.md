# Scene: Presentation

## Goal

帮助讲述者在有限时间内推动理解、说服或决策；幻灯片支持口头叙事，而不是替代讲稿。

## Required Input

演讲目标、受众、时长、核心结论、证据、演示环境、交付格式。

## Information Structure

1. **Opening**：主题、承诺、冲突或结论。
2. **Context**：为什么现在需要讨论。
3. **Core Argument**：3-5 个有顺序的论点。
4. **Evidence / Demo**：数据、案例、界面或过程。
5. **Decision / Takeaway**：请求、结论或下一步。
6. **Appendix**：方法、来源和细节。

## Recommendations

- Layout：`hero`、`two-column`、`comparison`、`timeline`、`grid`。
- Component：`quote-block`、`badge`、`workflow-node`、`code-block`、`card`。

## Constraints

- 一页一个主要结论，标题应表达结论而非主题名词。
- 字体、图表和截图按实际投影距离检查。
- 动画只用于控制披露顺序；静态导出仍可理解。
- 演讲备注承载细节，画面不堆完整段落。

## Common Errors

- 把文档段落直接贴到幻灯片。
- 所有页面使用同一标题加三卡片结构。
- 数据图没有单位、口径和结论。
- 结尾只有“谢谢”，没有明确决策或行动。

## Delivery Check

只看标题即可复述叙事链；每页视觉支持一个结论；结尾行动明确。
