# 质量检查清单

> 做完设计后逐条对照。P0必须全过，否则打回修改。

---

## P0（必须全过）

任何一条不过就要改：

- [ ] 色彩占比正确：纸色/中性色约90%，黄色约7%，编辑红约3%
- [ ] 没有使用禁忌清单里的任何元素（蓝紫渐变/glassmorphism/bounce动画/neon/居中病）
- [ ] 在开始制作前已经引导用户选择/确认设计风格、交付类型、排版方向
- [ ] 若任务同时包含展示 HTML 和卡片封面，已分别确认展示 HTML 标题字体与卡片封面标题字体，没有只问一次
- [ ] 背景使用灰暖纸张底（默认 #ECE7E2 或 #F4F1EC），非纯黑纯白，并带有极轻微纸张颗粒或纤维质感
- [ ] 视觉效果主要依赖字号、字重、留白、对齐和字体，不依赖大面积彩色块
- [ ] 没有粗黄色外框、整屏编辑红背景、大面积黄色卡片底，红色只作为批注/圈线/箭头/小编号点缀
- [ ] 卡片使用整体式信息卡，不是多层嵌套小框；内部层级靠留白、分区、细分隔线和局部批注建立
- [ ] 卡片圆角克制：主卡片约 22px，图片/顶部视觉区约 20px，小标签约 14px；没有过圆、过软、过可爱的卡片外观
- [ ] 卡片边框为 1px 低对比细线或更弱，阴影很弱或不用，没有重漂浮感
- [ ] 字体用了推荐字体池里的（黑体/汇文明朝体/霞鹜文楷/思源宋体/Arial Narrow/Helvetica Neue/Aptos Display/Source Serif 4/Libre Baskerville/Caveat等）
- [ ] 已确认用户选择的大标题字体：黑体 / 汇文明朝体 / 霞鹜文楷 / 思源宋体
- [ ] 已遵守字体搭配：黑体必须搭配汇文明朝体；霞鹜文楷和汇文明朝体在同一封面或首屏里只能二选一
- [ ] 已确认用户选择的正文模式：默认黑体 / 霞鹜文楷正文；若正文使用文楷，已检查长段落可读性
- [ ] 英文大标题默认使用稳定无衬线或低反差衬线，没有使用 Didot、Bodoni、Cormorant、高反差 Fraunces 或夸张 italic
- [ ] 手写/批注使用 Caveat，代码/终端/API Key/命令使用 Fira Code
- [ ] 思源宋体和汇文明朝体没有大面积滥用；霞鹜文楷用于正文时，按钮、导航、表格等功能文字仍保持清晰可读
- [ ] 有响应式（至少900px断点）
- [ ] 截图发Twitter不会被说"又是AI做的"
- [ ] 每个section布局形式不同
- [ ] clamp()做fluid sizing
- [ ] 没有使用任何HTML默认样式（默认blockquote、默认border-left引用、无样式ul/ol、默认table）——所有组件必须从components.md选用

---

## P1（应过）

尽量满足，提升品质：

- [ ] 至少一个section有视觉惊喜（出血排版/强字号对比/编辑式错位/克制装饰突破）
- [ ] 字号对比足够极端（大的>3rem，小的<0.85rem）
- [ ] 有Scroll Reveal动效 + stagger延迟
- [ ] 非封面内容区可使用大装饰数字或大透明英文做背景；封面页/首屏没有装饰性数字、水印数字或重复编号图形
- [ ] 有克制的高亮标记（短线、小圆点、手绘下划线或局部荧光笔），不使用左侧粗色条
- [ ] ::selection用强调色高亮

---

## P2（加分）

锦上添花：

- [ ] 图片溢出容器边界
- [ ] 有少量编辑红批注、圈线或手绘箭头打破节奏，且都指向具体信息
- [ ] 红圈没有悬空圈白；没有明确对象时已改用短下划线、小箭头或删除
- [ ] 装饰元素（手绘圈线/纸张颗粒/局部下划线）使用克制
- [ ] prefers-reduced-motion尊重
