# Brand DNA: AI Laboratory

## Identity

- **ID**：`ai-laboratory`
- **定位**：面向 AI 工具、Agent 与自动化工作流的实验室视觉系统，把复杂技术变成可检查的结构。
- **适用**：AI 产品、技术教程、Agent 介绍、API 文档、工作流、实验报告。
- **不适用**：情绪散文、婚礼活动、儿童内容、奢侈品叙事。
- **关键词**：精密、可验证、模块化、冷静、前沿但不赛博。

## Visual Character

像一张清晰的实验台：输入、处理、状态、结果都有明确边界和标记。技术感来自结构、数据和真实界面，不来自霓虹、满屏 monospace 或装饰电路。

## Color

| Token | Value | Semantic role |
|---|---|---|
| `--bg` | `#F1F3F0` | 实验室浅灰底 |
| `--surface` | `#FFFFFF` | 控制面与内容面 |
| `--ink` | `#121716` | 主要信息 |
| `--ink-muted` | `#5A6460` | 次级说明 |
| `--line` | `#CDD4D0` | 结构线与网格 |
| `--accent` | `#236B5D` | 主操作、成功路径、关键节点 |
| `--signal` | `#D2B13F` | 待处理、提示、选中 |
| `--danger` | `#B84B43` | 错误与风险 |

- 背景和中性色约 82%，绿色约 12%，黄与红合计不超过 6%。
- 状态色必须保持语义一致，不把红色当普通装饰。
- 深色模式使用 `#101513`，而不是纯黑；仅在终端、演示或用户明确要求时启用。

## Typography

- 中文标题与 UI：`Source Han Sans SC` / `Noto Sans SC`。
- 英文展示：`Aptos Display` / `Helvetica Neue`。
- 正文：无衬线；代码、变量、命令和模型名：`Fira Code`。
- 技术标识可使用等宽体，但标题和正文不能全站等宽。
- 数据与状态标签使用 tabular numerals，层级通过字重和尺寸建立。

## Graphic Language

- 采用 1px 结构线、节点、端口、连接线、坐标标记和分区编号。
- 工作流连接必须有方向，节点必须显示类型或状态。
- 图标使用线性几何图标；相同层级保持统一尺寸与线宽。
- 数据图表优先直接标注，少用独立图例；网格线保持低对比。

## Imagery

- 优先真实产品截图、局部放大的界面状态、流程图和数据可视化。
- 截图必须裁掉无关浏览器区域，并用注释指向操作位置。
- 插画采用抽象系统图、等距设备或精确线稿，不使用发光大脑、机器人头像和宇宙背景。
- AI 生成图只能用于概念氛围，不能代替真实产品状态。

## Motion

- 节点状态切换 120-200ms；流程演示 240-400ms。
- 允许一次性的路径描绘、数据刷新和状态脉冲。
- 禁止持续扫描线、无限发光、弹跳和无意义粒子。
- 复杂流程必须提供暂停或静态状态，并支持减弱动效。

## Decoration

允许坐标、实验编号、版本号、状态灯、测量刻度和轻量点阵。装饰必须模拟真实系统信息，不伪造不可解释的数据。

## Prohibitions

- 蓝紫霓虹渐变、玻璃拟态、发光边框、赛博城市。
- 用 monospace 和 `0/1` 背景假装技术感。
- 无方向连接线、不可解释节点、假的实时数据。
- 所有内容都塞入独立卡片。
- 只展示抽象插画，不展示实际工具或结果。

## Recommendations

- **Components**：`terminal-window`、`prompt-block`、`workflow-node`、`code-block`、`badge`、`annotation`。
- **Layouts**：`dashboard`、`step-flow`、`comparison`、`two-column`。
- 原因：输入、状态、过程与输出可以被清楚检查和复现。

## Acceptance

- 用户能快速区分输入、处理、状态和输出。
- 所有连接线有方向，所有状态色有固定语义。
- 技术感由真实数据、截图和结构产生。
- 代码与自然语言有明确字体和容器差异。
- 移动端流程能重排为单列且不丢失连接关系。
