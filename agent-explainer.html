AI Concept 01

Agent究竟
是什么？

为什么现在几乎所有 AI 产品都在说自己是 Agent？这次先把 模型、AI 产品、Agent、API放在同一张图里讲清楚。

模型 理解、生成、推理，像 AI 的大脑。
AI 产品 在模型外面加界面、文件、搜索和工具。
Agent 围绕目标，自主决定下一步，并调用工具执行。
01

第一步：先区分“模型”和“AI 产品”。

很多讨论混乱，是因为把底层模型和外层产品混在了一起。

Model

GPT、Claude、Kimi、DeepSeek，本质上是模型。

模型负责理解、生成和推理。它决定 AI 的基础能力，你可以把它理解成 AI 的“大脑”。

Product

ChatGPT、Claude 网页版、Kimi 网页版，是 AI 产品。

产品把模型装进可用的界面里，再加聊天、文件上传、联网搜索、工具调用等功能。

02

Agent 不是某个软件，而是一种 AI 的形态。

关键点不是它会不会聊天，而是它能不能围绕你的目标持续推进。

理解目标

先判断你真正想完成什么，而不是只回答一句话。

拆解任务

把目标拆成下一步、再下一步，形成行动顺序。

调用工具

搜索、读文件、写代码、跑终端、查数据库，都可以成为工具。

检查循环

看执行结果有没有问题，再继续修正和推进。

03

Chat 端和 Agent 最大区别：谁在执行。

普通聊天 AI 主要给你答案，Agent 会尝试帮你把事情往前做。

A

Chat 端：你问一句，它答一句。

它主要用文字、代码或建议来回应你，真正点击、复制、运行、检查的人通常还是你。

B

Agent：它先理解目标，再调用工具执行。

它会拆任务、调工具、检查结果、继续调整。执行链条变长了，AI 不只是“说”，也开始“做”。

04

具体产品怎么放？看它服务什么场景。

普通人不必把每个名字分得特别细，先看它是聊天、办公，还是写代码。

Workbot / Workbody 偏办公工作台，主要处理文档、表格、会议、流程等办公任务。
Tree 更像搭在模型之上的 AI 产品入口，能力取决于它接入的模型和工具。
Trae / Cursor AI 代码编辑器。能读项目文件、改代码、创建文件，也能调用终端。
Claude Code / Kimi Code / Codex 偏 Coding Agent。运行在终端或桌面里，围绕代码仓库执行任务、运行测试、修问题。
05

为什么现在所有产品都说自己是 Agent？

因为 AI 产品正在从“聊天框”变成“任务执行器”。

能看

它开始拥有上下文。

不只看你这一句话，还能看文件、项目、历史记录、当前任务和你的约束。

能做

它开始连接工具。

当 AI 能搜索、发邮件、读数据库、修改代码，它就更接近 Agent 的工作方式。

06

顺便解释一下 API：程序之间互相调用的接口。

如果不想订阅某个 Agent 产品的会员，就可能需要接入自己的 API Key。

API Key 可以粗略理解成一张“扣费凭证”。Agent 产品用它去调用模型、搜索、天气、邮件、数据库等服务，用多少就从你的额度里扣多少。它不该公开，也不要发给别人。

Conclusion

模型是大脑，产品是外壳，Agent 是会行动的形态。
