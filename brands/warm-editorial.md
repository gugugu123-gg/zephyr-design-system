# Brand DNA: Warm Editorial

## Identity

- **ID**：`warm-editorial`
- **定位**：像经过编辑与批注的个人刊物，用克制的纸面质感承载观点和叙事。
- **适用**：个人观点、成长文章、知识分享、访谈、书摘、轻量教程。
- **不适用**：高密度监控后台、赛博游戏、需要强烈促销感的电商活动。
- **关键词**：温暖、编辑感、可信、松弛、有人味。

## Visual Character

视觉应首先依靠排版、留白和内容节奏成立，纸纹与批注只用于增强“被认真编辑过”的感受。整体安静，但关键句有明确的人为痕迹。

## Color

| Token | Value | Semantic role |
|---|---|---|
| `--bg` | `#ECE7E2` | 主纸面 |
| `--surface` | `#F5F1EC` | 内层纸面与内容区 |
| `--ink` | `#171715` | 标题与主要内容 |
| `--ink-muted` | `#68625C` | 说明文字 |
| `--line` | `rgba(23,23,21,.12)` | 分隔与边界 |
| `--accent` | `#E8CB58` | 荧光笔、关键词 |
| `--signal` | `#A83D42` | 编辑批注、纠错、重点 |

- 中性色约 90%，黄色不超过 7%，编辑红不超过 3%。
- 同一信息块通常只使用一种强调色。
- 不使用纯白、纯黑或大面积高饱和色。

## Typography

- 中文标题：`LXGW WenKai`、`Source Han Serif SC` 或用户指定的授权字体。
- 中文正文：`Source Han Sans SC`；叙事内容可使用 `LXGW WenKai`，但 UI 控件保留无衬线。
- 英文展示：`Arial Narrow` / `Helvetica Neue`；英文编辑标题可少量使用 `Source Serif 4`。
- 批注：`Caveat`；代码：`Fira Code`。
- 标题和正文必须有明显级差；正文行宽建议 28-42 个汉字。
- 禁止同时堆叠文楷、明朝、宋体和手写体制造“设计感”。

## Graphic Language

- 使用细分隔线、页码、裁切标记、短下划线和有对象的手绘圈线。
- 图标采用简洁线性图标，线宽一致，不混用 emoji 充当功能图标。
- 数据图形使用墨色为主、单一强调色标重点。
- 圆角克制：内容卡片 8-16px；只有社交卡片可放宽到 22px。

## Imagery

- 优先真实照片、书页、桌面细节、人物局部和内容相关截图。
- 图片保留自然色，允许轻微降饱和与暖灰统一，不使用重滤镜。
- 插画应有手绘线条或版画感，避免通用 3D 人物和 stock AI 场景。
- 所有外部素材必须有明确来源或许可。

## Motion

- 进入动效：淡入加 8-16px 位移，180-320ms。
- 反馈动效：颜色、下划线或轻微位移，120-180ms。
- 长文滚动不使用持续漂浮或视差干扰阅读。
- 必须支持 `prefers-reduced-motion`。

## Decoration

允许纸张颗粒、真实内容旁的批注、短箭头、页码和局部荧光标记。每个装饰必须指向内容或建立层级，不允许悬空红圈、无意义胶带和重复水印数字。

## Prohibitions

- 蓝紫渐变、玻璃拟态、霓虹光、重阴影。
- 卡片嵌套卡片、全页同构网格。
- 大面积黄色或编辑红背景。
- 高反差时尚衬线、夸张 italic、无意义英文装饰。
- 默认浏览器样式直接出现在成品中。

## Recommendations

- **Components**：`quote-block`、`annotation`、`card`、`timeline`、`code-block`。
- **Layouts**：`hero`、`two-column`、`timeline`、`step-flow`。
- 原因：这些模式能让叙事、观点和人为批注成为视觉主角。

## Acceptance

- 去掉纸纹后，版面仍靠排版成立。
- 强调色面积符合比例，红色元素都有语义对象。
- 字体不超过三个角色：展示、正文、功能/代码。
- 连续 section 不重复同一构图。
- 截图中不存在通用 AI Landing Page 的蓝紫渐变和玻璃卡片。
