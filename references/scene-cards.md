# Scene: 小红书图文卡片（3:4）

> 适用于将文章、教程、观点拆解为小红书图文卡片。单个HTML文件包含所有卡片，支持一键导出PNG。

---

## 触发场景

用户说"做图文卡片"、"小红书图文"、"文章转卡片"、"帮我转成图文"、"做卡片"、"转成图文"等。

---

## 📐 卡片规格

| 属性 | 值 |
|------|-----|
| 尺寸 | 1080×1440px（3:4） |
| 最多张数 | 18张（小红书上限） |
| 文件形式 | 所有卡片放一个HTML文件，每张一个 `div.card` |
| 导出方式 | 顶部固定"一键导出PNG"按钮（html2canvas本地文件 + JSZip打包为zip一次性下载） |
| 浏览器预览 | `transform: scale(0.45); transform-origin: top center; margin-bottom: -792px` |

## 📝 排版原则

### 整体式信息卡原则

图文卡片默认使用整体式信息卡，而不是多层嵌套小框。卡片允许中等圆角，但必须克制，避免过圆、过软、过可爱。主要依靠留白、分区、细分隔线和局部批注建立层级，而不是依赖大量边框。

推荐数值：

| 项目 | 数值 |
|------|------|
| 主卡片圆角 | `22px` |
| 图片/顶部视觉区圆角 | `20px` |
| 小标签圆角 | `14px` |
| 边框 | `1px solid rgba(23,23,21,0.08)` |
| 阴影 | `none` 或非常弱，不要有重漂浮感 |
| 卡片内边距 | `32px ~ 40px` |
| 行与行之间 | `24px ~ 28px` |

硬规则：

- 卡片内部不能堆多个“小卡片”制造层级。
- 不用大量边框分割信息，优先用留白、分区、细线和局部批注。
- 不把圆角做成过软的可爱风；小标签除外，也控制在 `14px`。
- 信息卡要像一个完整的编辑版面，不像网页组件列表。

一句话标准：一个整体卡片，轻微圆角，克制边框，大留白，内部用分区而不是小卡片堆叠；有一点亲和感，但整体仍偏编辑感和信息感。

### 卡片设计语言

#### 1. 封面型卡片

适合一个主题、一句观点、视频或图文首屏、系列封面。

- 大标题作为主要视觉主体，允许主动换行。
- 使用大面积留白，不把信息铺满。
- 可加入一个小标签、编号、日期或系列名。
- 局部使用黄色或编辑红高亮，同一张卡只保留一个主要强调色；高亮不能承担大面积结构。
- 可加入一处手绘圈线或下划线，但必须和标题语义相关。

#### 2. 信息型卡片

适合一句观点加一段解释、方法论、知识点、信息摘要。

- 顶部放小标签、日期或编号。
- 中部放主标题，标题层级必须明显高于正文。
- 下部放简短正文，角落可放页码、来源或辅助信息。
- 文本块不能平均排列，要有明显主次。
- 信息密度适中，保持呼吸感。

#### 3. 图文型卡片

适合内容分享、案例展示、产品或工具介绍、过程说明。

- 上方放标题与标签。
- 中部放图片区域。
- 下方放正文、编号或说明。
- 图片与文字必须属于同一个信息系统。
- 不做成普通网页的图片卡片列表。

### 密度控制
- **内容少的页**（封面、金句、小结）→ 字巨大、大量留白、视觉冲击
- **内容多的页**（对比、列表、代码结构）→ 杂志排版，信息密度合理但不拥挤
- 每页都要适配画面，内容撑满但有呼吸感

### 变化感
- 每页排版要有变化——不能所有页都一样的layout
- 封面型 / 信息型 / 图文型 / 观点型穿插使用
- 大留白页和信息密度页交替

### 绝对禁忌
- ❌ 不要用左侧彩色竖线的卡片
- ❌ 不要用多余彩色圆点前缀
- ❌ 不要千篇一律列表（每项前面一个圆点/数字的平铺排列）
- ❌ 不要HTML默认blockquote样式
- ❌ 不要所有页都用同一种layout
- ❌ 不要制作参考网站的像素级复刻
- ❌ 不要做SaaS Landing Page、红色科技风官网、多Section产品介绍页
- ❌ 不要做功能卡片网格、导航栏、FAQ、用户评价和复杂Footer
- ❌ 不要过度渐变、阴影、发光和玻璃拟态
- ❌ 不要大面积黄色背景
- ❌ 不要无理由堆叠手绘元素
- ❌ 不要将复古字体作为通用设计捷径

## 🧩 卡片结构模板

### P1 封面
- 大标题（84px）可用霞鹜文楷（LXGW WenKai）或汇文明朝体（Huiwen Mincho），关键词优先用下划线、荧光笔或文字色强调，不使用大面积红色块。
- 副标题（44px）一行显示，紧跟标题下方，`white-space: nowrap`
- 圆形头像（`avatar.jpg`，120px），可用 1px 纸灰细边或极细黄色描边。
- 署名（作者名）44px + 介绍34px。
- 不使用整体黄色粗边框；可在角落放一个黄色小标记、短线或编号。
- 背景使用灰暖纸张色，叠加极轻微颗粒，不叠加网格纹理。
- 中间留白区域给用户放效果图/截图

### P2-Pn 内容页
- 根据内容量决定排版密度
- 每页选择不同的排版手法（见下方推荐列表）
- 页码编号用 oversized 淡色装饰数字

### 最后一页 尾页
- 金句（oversized引号装饰 `"` ，Fraunces 200px，opacity 0.15）
- 圆形头像 + 署名（作者名）
- CTA：「关注 + 作者名」（英文卡片可用你的固定英文 CTA 短语）
- 一行小字：你的签名档（见 `components.md` #43，中英文按卡片语言选择）
- 底部品牌双色装饰条

## 🎨 推荐排版手法

从V3验证通过的案例总结，按需组合：

| 手法 | 适用场景 | 要点 |
|------|----------|------|
| 极简封面 | 首屏、观点、标题页 | 巨大标题 + 强留白 + 顶部小标签 |
| oversized编号 | 步骤/流程展示 | 极大(64-120px)极淡色(opacity 0.12)做背景装饰 |
| 色块交替行 | 对比/表格 | 纸白 surface vs 柔纸色(#F4F1EC) 交替，不使用黄色底 |
| 大箭头流程 | 步骤连接 | 红色箭头(48px)连接流程块 |
| 代码面板 | 代码/文件树/命令 | 暖灰底+Fira Code+清晰缩进，不用深色终端底 |
| 金句装饰 | 核心观点/结尾 | oversized引号+米白底，装饰透明度低 |
| 问题→解法对比行 | before/after | 左侧暖灰、右侧编辑红轻强调，避免红绿对立 |
| 图标+文字横排 | 要点/优势列表 | 简单符号或小图标左对齐+文字说明 |
| 底部进度条 | 系列卡片 | 编辑红或黄色短条建立连续性，不做彩虹条，不超过画面宽度的 40% |

## 🔠 字体加载与检查

实现时必须：

- 优先调用本机字体，提供完整 fallback。
- 字体缺失时页面不能崩坏。
- 不依赖某一字体的精确字宽完成关键布局。
- 不提交来源不明或授权不明确的字体文件。
- Web Font 加载前先确认授权。
- 在浏览器开发者工具中检查 Rendered Fonts。

需要确认：

- 实际渲染为霞鹜文楷、汇文明朝体、思源宋体、Noto Serif SC、思源黑体或 Noto Sans SC 等预期字体。
- 没有意外回退到复古字体。
- 没有使用浏览器合成的伪粗体。
- Heavy / Black 字重真实存在；不存在时降低字重，不强行伪加粗。
- 中英文 fallback 没有导致断行和溢出异常。

## 🏷️ 品牌规范引用

- 所有颜色、字体、禁忌遵守 `brand-dna.md`
- 卡片正文、列表、说明文字必须使用黑体无衬线；霞鹜文楷可用于封面标题、核心观点句、情绪短句或少量批注，不用于长正文
- 头像源文件：`assets/avatar.jpg`（HTML内用相对路径 `avatar.jpg` 引用，交付时复制到输出HTML同目录）
- 署名固定为你在 `template-cards.html` 中配置的作者名（模板内为占位符，使用前替换）
- 品牌双色比例：编辑红 / 黄交替使用
- 背景主色：灰暖纸张 `#ECE7E2` / 柔纸 `#F4F1EC`，卡片 surface 用 `rgba(255,255,255,0.48)`；特殊深色面板用 编辑红 `#171715`

## 🌍 英文卡片补充规范

用于 Twitter/Instagram 等出海平台的英文卡片，在上述规范基础上调整：

- 标题/金句用 **Fraunces italic**（替代中文的 Noto Serif SC），手写标注用 **Caveat**
- 文案风格：除专有名词/缩写外全小写，句首字母大写
- 英文单词较长，标题字号可比中文规范上浮（封面可到 110-130px），注意单词换行不截断
- 尾页 CTA 用你的固定英文短语替代「关注 + 作者名」，签名档用英文版（见 `components.md` #43）
