# 暖纸编辑感

## 基本信息

- 风格名：暖纸编辑感
- 适合场景：科普解释、教程、观点内容、个人品牌页、知识型 landing、图文卡片
- 不适合场景：强促销电商页、重数据后台、赛博/霓虹/科技蓝大屏

## 视觉关键词

- 米白纸感
- 大标题
- 杂志式节奏
- 克制高亮
- 有人味但不幼稚

## 色彩规则

- 背景使用灰暖纸张色 `#ECE7E2`，局部浅区块用 `#F4F1EC`，卡片用 `rgba(255,255,255,0.48)`。
- 主文字使用墨黑 `#171715`。
- 高亮只使用黄色 `#F3D66F` 和编辑红 `#B63A3F`，且只用于局部文字、短线、标签、下划线、小圆点、手绘圈线和批注。
- 深色章节如确实需要，使用墨色 `#171715`，不要把编辑红做成大面积背景。
- 不使用网格背景、橙色、多色渐变、霓虹色、粗黄色外框、整屏编辑红或大面积黄色卡片底。

## 排版特征

- Hero 使用极大中文标题，允许中英文混排；英文大标题默认使用 Arial Narrow / Helvetica Neue / Aptos Display，低反差衬线只使用 Source Serif 4 / Libre Baskerville，禁止 Didot、Bodoni、Cormorant、高反差 Fraunces 和夸张 italic。
- 大标题必须在黑体、汇文明朝体、霞鹜文楷、思源宋体中选择，用于第一视觉记忆点。如果同时生成展示 HTML 和卡片封面，必须分别确认两者的标题字体。
- 黑体必须搭配汇文明朝体作为强调或副标题；霞鹜文楷和汇文明朝体在同一封面或首屏里只能二选一。
- 副标题可以使用思源宋体 / Noto Serif SC，保持短句和编辑感。
- 正文默认使用黑体无衬线；用户明确选择时可以切到霞鹜文楷正文模式，但导航、按钮、表格等功能文字仍优先黑体。
- 手写批注使用 Caveat，代码/终端/API Key/命令使用 Fira Code。
- 红色批注只用于指出信息。红圈必须圈住具体词、短语、数字、按钮或核心句，不能悬空圈空白；没有明确对象时改用短下划线、小箭头或删除。
- 思源宋体和汇文明朝体仅作为醒目强调，不大面积使用；霞鹜文楷可以作为用户选择的标题或正文模式。
- 图文卡片封面可以优先尝试霞鹜文楷，让标题更亲近、有手写感。
- 封面页/首屏只保留必要小标签文本，禁止装饰性数字、水印数字和重复编号图形；章节编号可以用于正文内容区，但不能放在封面当背景装饰。
- 标题可以用一小段编辑红或黄色高亮强调一个关键词，但不要使用大面积色块。
- 内容区使用非对称分栏、编号、卡片、引用、流程图建立节奏。
- 背景有轻微纸张颗粒或纤维质感，但不能抢文字注意力。
- 装饰线、标签、按钮只做小面积点缀。

## 禁忌

- 不要重新发明一套 landing page 风格。
- 不要用网格背景。
- 不要把编辑红和黄色铺满整屏或做成大面积面板。
- 不要使用黄色粗边框和黄色卡片底。
- 不要为了“设计感”堆太多组件。
- 不要把所有 section 都做成居中卡片。
