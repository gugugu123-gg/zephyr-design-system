# Audience: Technical Practitioners

- **背景**：开发者、AI 工程师、自动化实践者，能处理专业术语。
- **目标**：准确理解系统边界，复现步骤，判断结果是否可靠。
- **阻力**：伪代码冒充可运行代码、遗漏前置条件、状态和错误路径不清。

## Adaptation

- 可使用领域术语，但首次声明版本、环境和假设。
- 信息密度中高；摘要之后允许深入参数、代码和边界条件。
- 流程必须显示输入、处理、输出、失败状态和验证方式。
- 证据优先使用真实截图、命令输出、数据和可复现实例。
- Terminal Window、Code Block、Workflow Node、Badge 和 Annotation 优先。
- 代码可复制，颜色对比达标，不能只靠动画展示关键过程。

成功标准：读者无需猜测隐含步骤即可复现主要流程。
